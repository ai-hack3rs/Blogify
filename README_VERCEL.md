# Vercel Deployment Instructions

This project is configured for easy deployment to Vercel.

## Deployment Steps

1.  **Connect to GitHub/GitLab/Bitbucket**: Push your code to a repository.
2.  **Import to Vercel**: Create a new project in Vercel and import your repository.
3.  **Configure Environment Variables**:
    *   `GEMINI_API_KEY`: Your Google Gemini API key.
    *   `VITE_ADSENSE_CLIENT_ID`: Your Google AdSense Publisher ID (e.g., `1234567890123456`).
4.  **Deploy**: Vercel will automatically detect the Vite framework and deploy your app.

## Project Structure for Vercel

*   `vercel.json`: Handles SPA routing and API rewrites.
*   `api/`: Contains serverless functions (e.g., `/api/health`).
*   `dist/`: The build output directory (generated by `npm run build`).

## Note on Gemini API Key

The `GEMINI_API_KEY` is currently baked into the client-side code at build time. For production apps with high traffic, it's recommended to move Gemini calls to a Vercel serverless function in the `api/` directory to keep your key hidden from the browser.
